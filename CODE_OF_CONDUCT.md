# Code of Conduct

This project adopts the **[Contributor Covenant v2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/)** as its code of conduct. The full canonical text is published at the link above; this page summarises the parts you need to know to contribute here.

## TL;DR

We want this project to be a welcoming place for everyone — regardless of experience level, background, or identity. In practice that means:

- **Assume good faith.** Most disagreements are honest mistakes, not malice.
- **Critique code, not people.** "This logic is wrong because…" is fine; "you're wrong" is not.
- **Be patient with newcomers.** Everyone was new to FastAPI / Next.js / async Python once.
- **Stay on topic.** Issues and PRs are for project work, not unrelated discussion.

## What is not OK

- Personal attacks, harassment, slurs, or sustained disruption of discussions.
- Sharing other contributors' private information without consent.
- Sexualized language or imagery in any project space.
- Deliberately bad-faith engagement (sealioning, gaslighting, repeated boundary violations after being asked to stop).

The full list and definitions are in the [Contributor Covenant text](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).

## Reporting

If you experience or witness behaviour that violates this code, contact the maintainer privately:

**Email:** gaming.boy.fhx@gmail.com

We aim to acknowledge reports within 72 hours and to resolve or escalate them within two weeks. Reports are kept confidential to the extent possible. Retaliation against reporters is itself a violation.

## Scope

This code applies to all project spaces — the GitHub repository, issues, pull requests, code review, and any associated chat or forum — and to public spaces when an individual is representing the project.

## Enforcement

Maintainers may take any action they consider appropriate, up to and including a permanent ban from the project, in response to behaviour they deem to violate this code. Decisions are at the maintainer's discretion and are not subject to public debate; if you believe an enforcement action was unjust, raise it privately with the maintainer.
