Built to be customized

Everything a blog needs. Nothing it doesn't.

Minimal by design — a clean skeleton you dress up however you like.

Live Theme Switching

Paste any external CSS URL in the admin panel. Every visitor instantly sees the new look. No redeploy, no CDN flush — server-side injection on every page load.

Learn how it works →

Editor.js Rich Content

Block-structured editing with headers, ordered & unordered lists, inline code, fenced code blocks, images, embeds, and checklists. Content stored as portable JSON.

See the editor →

Role-Based Workflow

Three roles: Admin controls everything, Author writes after approval, Visitor reads without an account. Registration flows through admin approval.

See role matrix →

Admin Panel

Manage users, moderate all articles, review comments, switch themes, and configure site settings (name, logo, open/closed registration) — all from one dashboard.

Explore admin →

SQLite → PostgreSQL

SQLite with auto-schema creation for local development. Switch to PostgreSQL in production with a single DATABASE_URL env var. Alembic handles schema migrations.

Configure the DB →

Deploy Anywhere

Docker Compose (three services) for self-hosted setups. Or use the single-container Dockerfile — Nginx + FastAPI + Next.js + Supervisor — for Render and other PaaS providers.

Deployment guide →

Get up and running

From zero to published in minutes.

Three steps whether you're deploying locally or to the cloud.

01

Deploy

Run docker compose up locally or push to Render with the included single-container Dockerfile. The database schema is created automatically on first start.

02

Write

Register an account, get approved by the admin, then create rich articles using the Editor.js block editor. Toggle between draft and published with one click.

03

Style

Paste any CSS URL into the admin theme panel. Click "Activate". Every visitor instantly sees the new design — no redeploy, no cache invalidation needed.

Three roles, one platform

Built for every person on your team.

Each role has exactly the permissions it needs and nothing more.

Full control

The first user to register automatically becomes the admin, or you can set is_superuser = true in the database. Admins have complete control over the platform.

  • Approve or deactivate any user account
  • Edit or delete any article (not just own)
  • Manage and activate CSS themes from a URL
  • Configure site name, logo, and registration toggle
  • Moderate comments across all articles
  • View stats: total users, articles, and comments

Write & publish

Authors register via the public form and are activated by an admin. Once approved they can create and manage their own articles from a personal dashboard.

  • Create articles with the Editor.js block editor
  • Toggle articles between draft and published
  • Edit or delete their own articles
  • View per-article like and comment counts
  • Manage personal profile and biography
  • Cannot access other authors' articles

No account needed

Visitors can browse, search, read, and interact with published articles without an account. The reading experience is clean and distraction-free.

  • Browse all published articles on the home feed
  • Search articles by title (partial, case-insensitive)
  • Read articles with table of contents navigation
  • Like articles and leave comments (when logged in)
  • Register to become an author
  • Site look changes in real-time with active theme
